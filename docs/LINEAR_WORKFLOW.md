# Linear Workflow — Local Work OS

Linear is the source of truth for active work.

## Statuses

```text
Triage
Backlog

Assigned to Codex
PR Open
Review
QA
Done
Blocked

```

## Definition of Ready

An issue is Codex Ready only when:

```text
Goal is clear.
Scope is bounded.
Out-of-scope is explicit.
Relevant docs are linked.
Acceptance criteria are testable.
Testing requirements are listed.
Data model/activity/search/file impacts are identified.
Issue is small enough for one PR.
```

## Definition of Done

```text
PR merged to main.
CI passes.
Tests added/updated.
Manual QA documented where needed.
Linear issue moved to Done.
Follow-up issues created if needed.
```

## Import policy

Import tickets in waves:

```text
1. M0/M1 first.
2. M2/M3 when scaffold is stable.
3. M4–M6 for MVP completion.
4. M7–M14 only as later roadmap/backlog.
```
